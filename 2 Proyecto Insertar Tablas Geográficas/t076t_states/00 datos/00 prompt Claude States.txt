  Antes de los INSERTs de estados, verifica si el tipo ya existe.
  Genera el INSERT solo si NO existe en la tabla. Usa este patrón:
 
INSERT IGNORE INTO t075t_division_types
  (code, id_country, id_level_type, name_spanish, name_english, name_iso, 
   name_original, name_transcribed, status, created_at, updated_at, created_by, updated_by)
SELECT '[CODE]', [ID_COUNTRY], [ID_LEVEL_TYPE], '[ES]', '[EN]', '[ISO]', 
       '[ORIGINAL]', '[TRANSCRIBED]', 1, NOW(6), NOW(6), 1, 1
WHERE NOT EXISTS (
  SELECT 1 FROM t075t_division_types WHERE code = '[CODE]'
);
  

PASO 4 — GENERA los INSERTs de t076t_states:
  Para cada división de primer nivel del país genera un INSERT con estos valores:

  - geo_id            → ID numérico de geonames.org
  - id_country        → (SELECT id FROM t074t_countries WHERE code_iso_alpha2 = 'XX')
  - code              → código corto único, ej: CO-ANT
  - code_dane         → SOLO para Colombia; NULL para todos los demás países
  - code_dian         → SOLO para Colombia; NULL para todos los demás países
  - code_iso_numeric  → código numérico ISO 3166-2 de la división NO PUEDE SER NULO
  - code_iso_alpha2   → código alpha-2 ISO 3166-2, ej: CO-ANT
  - code_iso_alpha3   → código alpha-3 derivado, ej: COL-ANT
  - phone_code        → indicativo telefónico (heredar del país si no tiene propio)
  - id_division_type       → (SELECT id FROM t070t_division_type WHERE code = '[CODE]')
  - name_spanish      → Nombre en español',
  - name_english      → Nombre en inglés',
  - name_iso          → Nombre oficial ISO del país',
  - name_original     → Nombre en idioma oficial',
  - name_transcribed  → Nombre transcrito a alfabeto latino legible',
  - status            → 1
  - created_at        → NOW(6)
  - updated_at        → NOW(6)
  - created_by        → 1
  - updated_by        → 1

REGLAS

  - Usa utf8mb4; escapa comillas simples DENTRO de valores de texto
    duplicándolas (''), NO con barra invertida (\').
    Ejemplos de nombres que requieren escape:
      M'sila      → 'M''sila'
      Aïn N'Sour  → 'Aïn N''Sour'
      Oum el Bouaghi → sin apóstrofe, no requiere escape
    Aplica a TODOS los campos varchar: name_spanish, name_english,
    name_iso, name_original, name_transcribed, code, etc.
    Antes de generar el SQL, revisar todos los valores en busca de
    apóstrofes o comillas simples embebidas y duplicarlas.
  - NULL sin comillas para valores nulos
  - name_spanish en minúsculas con tilde si aplica
  - name_english siempre en inglés sin caracteres especiales
  - name_iso: nombre oficial exacto según ISO 3166-2
  - No inventes geo_id; si no encuentras el dato real de geonames.org, indícalo
  - Si el país tiene subdivisiones de distinto tipo (ej: regiones + provincias),
    agrupa por tipo y genera un INSERT de division_type por cada uno
